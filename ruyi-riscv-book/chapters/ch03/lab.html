实验 1：温控风扇

整合讲义 3.1–3.3 · 不重复 blink / uart-echo 单项练习

实验目的

讲义已分别练过 GPIO、串口日志思路、DHT22/开/关概念。本实验把它们组装成一条可演示闭环：

  • 周期性读取环境温度
  • 高于上限 → 风扇开；低于下限 → 风扇关
  • 读失败时打印错误并继续跑（不崩溃）
前置：讲义 3.1–3.3 跟做过；SSH 可用；课程套件含 DHT22、风扇 + 驱动模块、杜邦线。风扇由课程提供，学生不必自购。
与讲义的区分：不要再逐步抄 blink 的编译命令当正文。默认你会交叉编译与 scp。本页只写接线检查、temp-fan 工程、验收标准。

L1.2–L1.4 构建、运行与观察

1. 编译部署

默认工程 SIMULATE_SENSOR=1：无 DHT 也可演示阈值开/关逻辑。接好套件后改为 0 再编译。用户态读 DHT22 可能偶发失败——程序应打 [ERR] 并继续，勿崩溃。

$ cd chapters/ch03/code/temp-fan
$ make clean && make
$ file temp-fan          # 应为 RISC-V ELF
$ scp temp-fan user@board-ip:~/
$ ssh user@board-ip './temp-fan'

2. 预期行为

  • 终端周期性打印温度（及湿度，若实现）。
  • 用手捂传感器或靠近热源使温度升过 T_HIGH → 风扇转。
  • 移开冷却至低于 T_LOW → 风扇停。
  • 故意松 DATA 线一次 → 出现 [ERR] 类日志，程序不退出，恢复接线后继续采样。

3. 阈值与滞回

使用 T_HIGH / T_LOW 两档（滞回），避免温度在临界值附近风扇狂抖。默认值可在源码宏中改；改完重新编译部署。

运行中勿带电拔插电源线。演示加热时注意模块与手指安全，勿短路引脚。
