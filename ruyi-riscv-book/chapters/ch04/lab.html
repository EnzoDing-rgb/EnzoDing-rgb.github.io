本章实验 · 串口命令温控

自动控温同时敲 status / set · 接讲义 4.1–4.3

说明

讲义已讲清串口字节进出、命令表与 select。本实验在脚手架上补全两块，并在荔枝派 4A 上演示：

  • 自动按阈值开/关风扇（与第三章闭环一致）；
  • 同时在终端敲 status / set，立刻得到回复，且新阈值马上参与控温。
前置：讲义 4.1–4.3；第三章温控接线可用；RuyiSDK 交叉工具链与 SSH 可用。工程：chapters/ch04/code/serial-thermo/。
你要改的代码：main.c 里标了 TODO 的命令处理函数、命令表注册，以及 select 主循环。不要删掉已有的采样与风扇逻辑。

步骤

1. 核对宏与接线

打开 main.c，确认 GPIO_CHIP_PATH、DHT_LINE、FAN_LINE 与课程接线表一致。默认可先用 SIMULATE_SENSOR=1 演示阈值与命令，再改 0 接真 DHT22。

2. 补全命令表

  1. 实现 cmd_status：打印温度、湿度、风扇状态、t_high/t_low。
  2. 实现 cmd_set：解析 set high <数> / set low <数>；校验 low < high 后写入状态结构。
  3. 把两条命令挂进 g_cmds[]，用查找函数按名字分发。

3. 补全 select 主循环

  1. 用 select 同时等命令输入描述符与采样超时。
  2. 超时 → 调用已有采样与阈值控温。
  3. 可读 → 读一行、查表执行；未知命令打印简短错误，进程不退出。

4. 编译部署

$ cd chapters/ch04/code/serial-thermo
$ make clean && make
$ file serial-thermo
$ scp serial-thermo user@board-ip:~/
$ ssh user@board-ip './serial-thermo'

5. 联调顺序（固定）

  1. 启动后观察周期日志（温度 / 风扇）。
  2. 敲 status，核对打印字段完整。
  3. 敲 set high 30，再 status 确认上限已变。
  4. 在模拟升温或捂热过程中，确认风扇仍按阈值动作，且打字回复不卡住采样。
