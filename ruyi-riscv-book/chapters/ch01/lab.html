环节一 · 接 1.1（平台与工具链）

前提：你已按讲义安装 ruyi、验证交叉工具链。

任务 A：平台分层与个人清单

  • 自绘分层图（ISA → SoC → 开发板 → OS → 工具 → 应用），标出你课程中的具体实例。
  • 在板卡资料上标注电源、串口、网络、启动介质接口，形成个人环境清单（硬件/线缆/主机软件各一项状态）。

任务 B：主机 gcc vs 交叉 gcc（对比实验）

同一源文件 mini.c（return 0 即可），分别编译：

$ gcc mini.c -o mini-host && file mini-host
$ riscv64-unknown-linux-gnu-gcc mini.c -o mini-riscv && file mini-riscv

用一两句话说明：为何不能把 mini-host 直接 scp 到 LicheePi 运行？（呼应讲义 1.1 课堂练习。）

任务 D：排错一题（动脑）

再答：若「批量传 20 个文件到板子」，优先 SSH/SCP 还是串口？一句话理由。

环节三 · 接 1.3（一键准备决策）

前提：已读讲义 1.3 六维对比与六项风险。

任务 E：对你这块 LicheePi 做决策

  1. 运行 ruyi device --help 及一键准备相关帮助（以本机为准），记录：是否列出你的板卡/变体。
  2. 填写六项风险检查表（是/否/不适用 + 一句备注）。
  3. 写出结论（三选一并说明理由）：
    • 继续用手工烧录（§1.2）
    • 可尝试一键准备（仅在六项全过、空白介质时）
    • 当前 ruyi 不支持 — 说明依据与手工路径
安全：本实验不强制真实执行写入。做出有依据的「不执行一键准备」同样完成实验。

任务 F：场景判断（简答）

讲义 1.3 三个场景（有重要数据 / 设备未列出 / 空白介质），各用一句话说明你会选手工还是一键准备。
