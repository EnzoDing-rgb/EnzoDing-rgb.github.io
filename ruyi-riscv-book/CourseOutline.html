讲义与本章实验

文档 解决什么问题 学生要交出什么
讲义（lecture） 零件是什么、接口怎么用；每章固定三节；带跟做小例子；每章有一节深入理解 跟做后单项能力会了，例如灯会闪、会打一行串口日志
本章实验（lab） 用本章零件做一场完整实验；给 C 脚手架 + 规格 + 验收标准 能演示约定现象，例如捂热风扇转，并填简单验收记录
  • 每章讲义固定三节（x.1 / x.2 / x.3）；本章实验是一整块。
  • 实验正文写差量步骤和验收标准；安装 / make / scp 等基础操作以讲义为准。
  • 全文直接写「做什么、看到什么」；每章深入理解写死一种做法，步骤人人相同。

术语表

读章节总览前可先扫一眼；各章细则表内仍有章节专属名词。每个词用一句话说清「它是什么」。

术语 说明
荔枝派 4A 本课用的 RISC-V 开发板；板上已跑 Linux，用 SSH 或串口连上去写程序。
RuyiSDK / ruyi 如意课题组的 RISC-V 开发工具入口：装交叉编译器、系统镜像、查板卡支持列表。
交叉编译 在电脑上编出只能在开发板上跑的程序，再拷到板子上运行。
SSH / SCP / 串口 SSH：远程登录板子敲命令。SCP：把文件拷到板子。串口：USB 线连上后的「聊天线」黑窗口，打字板子能看见。
CoreMark 测板子算力的小程序。ch01 实验：交叉编译后上板跑出分数并记录。
GPIO 板子上一排针脚。程序可读（按键、传感器）或可写（灯、继电器）。
gpiod Linux 上读写 GPIO 针脚的常用接口库；本课用它控灯和风扇继电器。
MMIO 把硬件寄存器当成内存地址来读写。Linux 里内核管着这些地址；用户程序经 gpiod 等接口间接访问。ch03 用只读方式亲眼看某一位翻了。
devmem 在板子上按地址读/写内存的小工具。ch03 只读：点灯前后各读一次，看到那一位从 0 变成 1。
DHT22 温湿度探头物料。插在 GPIO 针脚上；程序问它「现在几度」，得到温度数字。
继电器 用小信号开关大电流的模块。板子 GPIO 驱动继电器，继电器再给风扇通断电。
阈值控制 温度高于某值开风扇，低于某值关。数字可写在程序里，ch04 起可用串口命令现场改。
串口命令 你在串口黑窗口打字，板子认命令。例：status 查状态，set 30 把阈值改成 30。
select 让程序同时盯几个输入源。ch04：一边等 DHT22 采样周期到，一边等你打字进串口。
MQTT 物联网发消息的方式。手机和板子都连一个「中转站」，按主题名收发文字（如主题 home/led 发 on）。
Broker MQTT 的中转站。常用软件叫 Mosquitto，可跑在电脑或板子上。
发布 / 订阅 发布＝往某主题发一条消息；订阅＝盯着某主题收消息。ch05：手机下行控外接 LED；板子上行发布状态。
tcpdump 抓网络包的工具。ch05：抓一条 MQTT 发布，亲眼看到消息被一层层包起来发出去。
外接 LED 远程控制用的灯。ch05：手机发 MQTT 命令，板子驱动这盏灯。
告警红灯 本地异常指示灯（板载或外接）。综合项目里规则或模型判异常时点亮。
pthread / 多线程 一个程序里同时跑多段逻辑。ch06 三线程：采集 ∥ 控制 ∥ 通信(MQTT)。
竞态 两条线同时改同一份数据，结果写乱。ch06：先故意不加锁看数据错乱，再加锁修好。
ThreadSanitizer 编译时加上 -fsanitize=thread 的检查器。ch06：在电脑上编同一个抢数据小程序，终端打印哪两行在抢同一变量。
规则/模型仲裁 超阈值 或 模型判异常 → 风扇转；日志记下是谁触发的。综合项目的智力核心。
老师供权重 / 学生做推理 老师用电脑训好 tiny 模型，课里发 weights.h；学生在板上写手写 tiny 前向，接到风扇/灯/MQTT。
weights.h 模型权重写成 C 头文件，编译进板子程序；推理时读这些数字做乘法相加。
智能环境终端 本课综合大项目：复用 ch06 三线程；外接灯远程控、告警红灯本地异常；风扇用规则与模型一起判；老师供权重 + 手写 tiny 前向。
