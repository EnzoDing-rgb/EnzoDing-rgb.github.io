本章实验 · MQTT 远程控灯

下行控外接 LED · 上行状态 · 抓一条 publish · 接讲义 5.1–5.3

说明

讲义已覆盖 Broker / 主题 / 发布订阅、下行控灯与 tcpdump 分层。本实验在荔枝派 4A 上完成可演示闭环：

  • 订阅 course/led/cmd，用 on/off 驱动外接 LED；
  • 状态变化后 publish 到 course/led/status；
  • 按讲义步骤抓一条 publish，能指出 MQTT 在 TCP/IP 之上。
前置：讲义 5.1–5.3；Mosquitto（或课堂 Broker）可达；外接 LED 接线与 LED_LINE 一致。工程：chapters/ch05/code/mqtt-led/。
你要补全：订阅回调里的点灯逻辑；publish_status 桩。连接与循环骨架已给出。

步骤

1. 配置宏

编辑 main.c：BROKER_HOST（主机局域网 IP，勿用板端 127.0.0.1）、TOPIC_CMD / TOPIC_STATUS、GPIO_CHIP_PATH、LED_LINE。

2. 补全下行与上行

  1. 在消息回调中：payload 为 on/off 时设置 GPIO，并调用 publish_status。
  2. 实现 publish_status：向状态主题发布当前灯态字符串。
  3. 未知 payload 打印 [ERR]，不改电平。

3. 编译部署

$ cd chapters/ch05/code/mqtt-led
$ make clean && make
$ file mqtt-led
$ scp mqtt-led user@board-ip:~/
$ ssh user@board-ip './mqtt-led'

4. 联调（固定顺序）

  1. 主机：mosquitto_sub -h BROKER_IP -t 'course/led/status' -v
  2. 主机：mosquitto_pub -h BROKER_IP -t 'course/led/cmd' -m on → 灯亮，订阅端见到状态。
  3. 再发 off → 灯灭，状态同步。
  4. 手机 MQTT App 连同一 Broker，重复 on/off 一次。
  5. 按讲义 5.3 用 tcpdump -i any -nn -A port 1883 -c 20 抓一次发布，保存输出。

验收

项通过条件
编译交叉编译成功，file 为 RISC-V
下行pub on/off 外接 LED 亮灭正确
上行状态主题订阅端能看到对应 payload
手机联调App 发命令灯有动作（与主机 pub 等价）
抓包保存含 1883 的 tcpdump 片段，能口述 MQTT→TCP→IP
完成标志：上表五项齐全；可进入第六章把 MQTT 放进独立通信线程。
