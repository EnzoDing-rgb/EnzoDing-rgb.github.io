本章实验 · MQTT 远程控灯

下行控外接 LED · 上行状态 · 抓一条 publish · 接讲义 5.1–5.3

说明

讲义已覆盖 Broker（消息中转站）、主题、发布/订阅、下行控灯，以及用 tcpdump 看「MQTT 装在 TCP 里、TCP 装在 IP 里」的分层。本实验在荔枝派 4A 上完成可演示闭环：

  • 板端程序订阅命令主题 course/led/cmd，收到载荷 on / off 后驱动外接 LED；
  • 灯态变化后，向状态主题 course/led/status 发布当前状态；
  • 按固定步骤抓一条 publish，能指出 MQTT 运行在 TCP 端口 1883 之上。
前置：已读讲义 5.1–5.3；Mosquitto（或课堂统一 Broker）在局域网可达；外接 LED 接线与源码宏 LED_LINE 一致。工程目录：chapters/ch05/code/mqtt-led/（源码 main.c，说明见 README.md）。板端需已安装/可用 libmosquitto 与 libgpiod（按课堂环境配置）。
你要补全：消息回调 on_message 里的点灯逻辑；以及 publish_status 函数桩。连接 Broker、订阅命令主题、以及 mosquitto_loop_forever 循环骨架已给出，不要推倒重写。

步骤

1. 配置宏

编辑 chapters/ch05/code/mqtt-led/main.c 顶部宏：

  • BROKER_HOST：填 Broker 的局域网 IP（主机跑 Mosquitto 时，用主机在局域网中的地址）。不要写成板端的 127.0.0.1——那只指向板子自己，连不到主机上的 Broker。
  • BROKER_PORT：默认 1883（MQTT 明文常用端口）。
  • TOPIC_CMD / TOPIC_STATUS：默认 course/led/cmd 与 course/led/status；若课堂统一改了主题，以板书为准并保持发布端与订阅端一致。
  • GPIO_CHIP_PATH、LED_LINE：与外接 LED 接线表一致。

本步成功判据：宏已按课堂环境改好；你能说出 Broker IP 与两个主题名。

2. 补全下行控灯与上行状态发布

术语提醒：下行指从手机/主机发命令到板子控硬件；上行指板子把当前状态发回 Broker，供别人订阅。

  1. 在 on_message 中处理命令载荷
    脚手架已把 payload 拷进字符串 buf。你要做：若精确等于 on，调用已有 led_set(1)，再调用 publish_status("on")（或与你约定一致的状态字符串）；若等于 off，则 led_set(0) 并 publish_status("off")。
    其他任意字符串：打印 [ERR]，不改 GPIO 电平。
    成功样子：主机 pub on 后板端日志有收包记录，外接 LED 亮；pub off 后灯灭。
  2. 实现 publish_status
    用 mosquitto_publish 向 TOPIC_STATUS 发布当前灯态字符串。成功打 [INFO]，失败打 [ERR]（可参考 mosquitto 返回值与 mosquitto_strerror）。
    成功样子：另一台机器上 mosquitto_sub -t course/led/status -v 能看到与灯态一致的 payload。

3. 交叉编译并部署

$ cd chapters/ch05/code/mqtt-led
$ make clean && make
$ file mqtt-led
# 应为 RISC-V ELF
$ scp mqtt-led user@board-ip:~/
$ ssh user@board-ip './mqtt-led'

启动后应看到正在连接 Broker、连接成功、以及已订阅命令主题的日志。若连接失败，先检查 BROKER_HOST、防火墙是否放行 1883、板子与主机是否同一网段。

4. 联调（固定顺序，全部要做）

  1. 在主机开一个订阅窗口，盯住状态主题： 
    $ mosquitto_sub -h BROKER_IP -t 'course/led/status' -v
  2. 再开一个窗口发下行命令： 
    $ mosquitto_pub -h BROKER_IP -t 'course/led/cmd' -m on
    期望：外接 LED 亮；订阅窗口出现状态 payload（如 on）。
  3. 再发 -m off：灯灭，状态同步为 off。
  4. 用手机 MQTT 客户端连同一 Broker，对同一命令主题再发一次 on/off，确认与主机 pub 等价（灯有动作）。
  5. 抓包（必做）：在主机或板端（以讲义 5.3 与课堂约定为准，通常在能看到板↔Broker 流量的那一侧）执行： 
    $ sudo tcpdump -i any -nn -A port 1883 -c 20
    然后触发一次 publish（例如再 pub on，或让板端上报状态）。
    你应当在 tcpdump 输出里看到什么：
    • 过滤条件 port 1883 命中的包（MQTT 默认明文端口）；
    • 用 -nn 看到的 IP 地址与端口，对应「这是 TCP 连接上的流量」；
    • 用 -A 打印的 ASCII 区，有时能瞥到主题名或 on/off 一类载荷片段。
    分层心智（口述即可）：应用层是 MQTT 的 publish 内容 → 装在传输层 TCP（端口 1883）里 → 再装在网络层 IP 里发出去。tcpdump 让你亲眼看到端口与载荷痕迹，而不是只相信库函数返回成功。把含 1883 的片段保存下来作为验收证据。
