实验目的

讲义已覆盖类型、函数、指针与结构体。本实验给你一份故意有坑的假温控脚手架，按关卡改到行为符合下表。

  • 脚手架：chapters/ch02/code/fake-thermostat/
  • 输入：内置温度数组，或从 stdin 读浮点数；无真实 DHT 传感器
  • 输出：每轮打印温度与风扇 ON/OFF 决策
前置：读完讲义 2.1–2.3；本机有 gcc 与 make。优先本机编译，专注改 C；可选交叉编译上板自验。
不要一上来大段重写。先跑通半成品，对照关卡表，一次改一类 bug。

关卡表

默认配置：t_high = 28.0，t_low = 26.0，滞回：高于上限开、低于下限关、中间保持。

关卡现象（修好前）你要改到主要坑类型
L1 温度升高时风扇反而关，或该开不开 比较方向正确：temp > t_high → ON；temp < t_low → OFF 错误比较
L2 阈值「怎么改都不对劲」，行为像用错了字段 决策使用 cfg->t_high / cfg->t_low，不要误用 period_ms 结构体字段用错
L3 传入空配置或空风扇指针时崩溃 / 未定义行为 指针为 NULL 时安全返回，不解引用 指针使用
L4 传感器读失败仍用垃圾温度做决策 读失败时打印错误并跳过本轮决策，保持上一风扇状态 缺少错误分支

内置序列验收参考

修复全部后，默认内置序列的期望决策（初始风扇 OFF）：

构建与运行

$ cd chapters/ch02/code/fake-thermostat
$ make                 # 默认本机 gcc
$ ./build/fake-thermo  # 跑内置序列

# 可选：从 stdin 喂温度（一行一个浮点数；负数表示读失败）
$ printf '25\n27\n29\n27\n24\n-1\n' | ./build/fake-thermo --stdin

# 可选：交叉编译
$ make CROSS_COMPILE=riscv64-unknown-linux-gnu-

细节与修好后的期望说明见脚手架内 README.md。

验收清单

  • L1–L4 对应逻辑均已修正，程序可稳定跑完内置序列；
  • 输出与「内置序列验收参考」一致（含读失败保持状态）；
  • 能口头指出半成品里至少三类坑：比较、字段、指针或错误分支；
  • 无需真 DHT、无需上板即可完成本章验收（上板为加分项）。
