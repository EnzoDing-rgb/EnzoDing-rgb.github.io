本章实验 · 温控风扇

DHT22 + 风扇 + 继电器 · chapters/ch03/code/temp-fan/ · devmem 只读核对

说明

讲义已建立 I/O 心智、练过 GPIO 闪灯，并理清温控数据流与寄存器位翻转。本实验组装成一条可演示闭环，并完成 devmem 只读核对。

  • 脚手架：chapters/ch03/code/temp-fan/
  • 硬件：DHT22、风扇、继电器（用小信号开关风扇大电流）、杜邦线；荔枝派 4A
  • 行为：周期读温；高于上限开风扇；低于下限关风扇；读失败打 [ERR] 且程序继续跑
前置：讲义 3.1–3.3 跟做过；SSH 可用；课程套件已到手。风扇由课程提供。
GPIO 不直接驱动风扇电机——只接继电器信号端。运行中勿带电拔插电源线。

步骤

1. 接线

按下表连接；具体脚位以当堂接线表为准，并与 chapters/ch03/code/temp-fan/main.c 中的宏一致。

模块接到哪里注意
DHT22VCC、GND、DATA→某 GPIODATA 按模块说明是否需上拉；对应宏 DHT_LINE
继电器 → 风扇信号端→某 GPIO；风扇电源按套件说明对应宏 FAN_LINE；共地必接
共地板与模块 GND 相连漏接 GND 是常见故障

在源码中确认：GPIO_CHIP_PATH / DHT_LINE / FAN_LINE / T_HIGH / T_LOW / SAMPLE_MS / SIMULATE_SENSOR。默认示例：T_HIGH=28.0、T_LOW=26.0、SAMPLE_MS=2000。

2. 构建与运行温控闭环

先把 SIMULATE_SENSOR 设为 1 验证阈值逻辑；接好 DHT22 后改为 0 再编译部署。

$ cd chapters/ch03/code/temp-fan
$ make clean && make
$ file temp-fan          # 应为 RISC-V ELF
$ scp temp-fan user@board-ip:~/
$ ssh user@board-ip './temp-fan'

预期行为：

  • 启动时打印阈值与采样周期；模拟模式下有 SIMULATE_SENSOR=1 提示。
  • 终端周期性打印 [INFO] temp=… hum=… fan=ON|OFF。
  • 模拟升温越过 T_HIGH，或用手捂传感器使真温度升过上限 → [INFO] fan ON，风扇转。
  • 温度降到低于 T_LOW → [INFO] fan OFF，风扇停。
  • 真机模式下故意松 DATA 线一次 → 出现 [ERR] …，进程不退出；恢复接线后继续采样。

3. devmem 只读核对寄存器位

在温控闭环之外，用讲义 3.3 的方法完成一次对照（地址与 bit 以当堂下发为准）：

  1. 记录点灯（或控继电器）前：sudo devmem 0xADDRESS 32 的读数。
  2. 用 gpiod 将该脚置高（blink 一次，或 temp-fan 触发 fan ON）。
  3. 再读同一地址，确认对应 bit 已翻转。
  4. 把前后两个十六进制值与「哪一位翻了」写入记录表。全程只读，不写寄存器。
