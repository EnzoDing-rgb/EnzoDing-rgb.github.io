讲义 vs 本章实验（怎么区分）

两套文档，相关但不重复。整门课只有一条学习顺序，没有选修分叉。

文档 解决什么问题 学生要交出什么
讲义（lecture） 零件是什么、接口为啥这样用；每章固定三节；带跟做能跑通的小例子（单项，不替代实验） 跟做后，单项能力会了（例如灯会闪、会打一行串口日志）
本章实验（lab） 用本章零件打一场有胜负条件的完整仗；半成品 + 规格 + 硬验收；默认你会讲义里的基础操作 能演示约定现象（例如捂热风扇转），并填简单验收记录
  • 每章讲义只切三节（x.1 / x.2 / x.3）；本章实验是一整块，可有小标题，不再拆成多个「实验环节章」。
  • 实验正文不重贴讲义里的安装 / make / scp 教程，只写差量步骤和验收标准。
  • 不再使用全局「实验一」「实验二」编号；温控风扇 = ch03 本章实验，远程控灯 = ch05 本章实验。

术语表

读「项目递进」前可先扫一眼；各章小节表内仍有章节专属名词。

术语 说明
MQTT 物联网用的发消息协议。手机、开发板不直连，都连一个「中转站」，按主题名收发文字命令（如 home/led 发 on）。
Broker MQTT 的中转站。谁发布、谁订阅，都由它转发。常用 Mosquitto 在 PC 或板子上跑。
发布 / 订阅 发布 = 往某主题发一条消息；订阅 = 盯着某主题，有新消息就收到。ch05 本章实验：手机发布、板子订阅。
主题（topic） 消息频道名，像微信群名。例：home/led 控灯，sensor/alert 报告警。
RuyiSDK / ruyi 本课统一的 RISC-V 开发工具入口：装交叉编译器、系统镜像、查板卡支持列表。
交叉编译 在 PC 上编译出只能在开发板上跑的程序，再 SSH/SCP 拷过去运行。
一键准备 / 一键编译 RuyiSDK 提供的自动化能力（如按支持列表写镜像、便捷构建）。讲义必讲（推广 ruyi）；本章实验不强制用一键准备烧录，手工烧录可过。
CoreMark 嵌入式常用算力基准。ch01 章末讲义介绍；本章实验硬性验收：交叉编译后上板跑出分数并记录。
GPIO 板子引脚，软件可读（按键、传感器）或可写（LED、继电器）。本课从接口用法理解，不讲芯片寄存器手册。
MMIO 内存映射 I/O：用访存方式访问设备。RISC-V 常用此方式；Linux 下由内核驱动管理，用户态经 API / 设备节点访问。
DHT22 温湿度传感器。板子定时读它，得到当前温度、湿度数字。
PWM 用快速开关调节平均电压，用来控风扇转速或 LED 亮度（讲义概念；温控实验主线是开/关）。
阈值控制 温度高于某值开风扇，低于某值关。可写在程序里，ch04 起改到配置文件。
pthread / 多线程 一个程序里同时跑多段逻辑（如一线程读温度、一线程处理 MQTT），互不卡死。
时序异常检测 不看单次温度，看最近一串读数（滑动窗口）像不像「正常环境」。不像 → 判异常。综合项目的推理部分就是这个。
老师供权重 / 学生做推理 老师用 PC/GPU 训好 tiny 模型，课里发 weights.h；学生在板上写手写 tiny 前向并接到风扇/灯/MQTT。学生本阶段不自己训练。
weights.h 模型权重的 C 头文件，编译进板子程序，推理时直接读这些数字做乘法相加。
智能环境终端 本课综合大项目：测温、控风扇、MQTT 控灯/上报，并用小模型判断环境是否异常。

归属 名称 你要看见的现象 串起的能力 一句话
ch01 本章实验 上板通道验收 交叉编译 CoreMark → SCP 上板 → 跑出分数并记录；SSH/uname 自证通道通 ruyi、交叉编译、Makefile、部署 通道硬验收；不强制一键准备烧录。

ch05 本章实验 MQTT 远程控灯 手机发消息 → 板上 LED 亮灭；板子可回传状态 网络、MQTT、GPIO 输出 远程控制；与温控故事分开验收。
ch06 本章实验 多线程协同 采集∥控制同时跑，捂热/模拟升温风扇仍按阈值转停；Ctrl+C 后关风扇并干净退出 pthread、共享状态保护、硬件收尾 双线程温控骨架；不写 Shell。
综合项目 智能环境终端 异常时：红灯 + 风扇 + MQTT 告警几乎同时发生 总装 + 老师权重 + 手写前向 组装前面各章，加上边缘推理部署。

硬课规

  • 板载应用用 C 语言（MQTT 可用 libmosquitto 等 C 库）
  • 用 RuyiSDK 获取工具链与目标环境；讲义覆盖一键准备 / 一键编译（实验不强制一键写入）
  • 必须走通 交叉编译 + Makefile（或等价模板）；ch01 以 CoreMark 上板跑分作通道硬验收
  • 必须有 综合大项目 可演示成果
  • 模型权重由老师提供；学生做板端手写 tiny 前向与部署；不在板上训练；不以大模型板上推理为主线
  • 原理讲到接口层（会用 fd / gpiod / socket 等）；需要时交代 MMIO 等心智；不讲成操作系统内核课，不讲成 RISC-V 指令集课
  • 一条线顺序学；不设选修双轨；每章讲义固定三节 + 一个完整本章实验
  • 实验默认给 C 脚手架，学生按讲义往后写；验收必须结合板上硬件可见现象；不把 Shell/作业控制当主线（有别于 CSAPP 式系统软件课）
  • OpenCV、YOLO、Buildroot 等不进主线

训练与推理分工

  • 老师（课前准备）：在 PC/GPU 上训好 tiny 模型，发布 weights.h（可附带说明与对照数据）
  • 学生（主线）：滑动窗口 → 手写 C 前向 → 正常/异常 → 风扇 + LED + MQTT
  • 课堂重点：权重如何进工程、前向如何嵌进多线程、如何与规则控制一起工作
