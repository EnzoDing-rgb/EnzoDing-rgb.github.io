本章实验 · 温控风扇

从零拼装：DHT22 + 继电器 + 风扇 + 电平转换 · 荔枝派 4A

这节课要做成什么样

用一句话：板子周期读温度；温度高了风扇转，温度低了风扇停；传感器线松了程序还继续跑，只打印出错信息。

你不需要先成为电子工程师。本页按「认零件 → 认排针 → 一根根接线 → 改程序里几个数字 → 编译上传 → 看现象」写。每出现一个专有词，都会先用人话解释。

成功时你能演示：终端里周期性打印温度；温度升过上限出现风扇开、风扇真转；温度降下来风扇停；故意松传感器数据线，出现带 [ERR] 的一行字，程序不退出。
安全：风扇的大电流不要直接接在板子的「信号脚」上——必须经过继电器。板子上标着 Fan Power 的白色插座是板载散热风扇用的，跟本实验风扇无关，别插错。接线时尽量断电操作。

桌上要摆什么

东西它干什么（人话）
荔枝派 4A跑 Linux 的小电脑；右边一排针用来接线
DHT22 温湿度模块测温度（和湿度）；三根针：电源、地、数据线
继电器模块（1 路，5V）用小信号扳动的「自动开关」，去通断风扇电源
5V 小风扇被继电器开关的负载；红线正极、黑线负极
TXS0108E 小板信号翻译器：板子脚约 1.8V，模块侧约 3.3V，中间靠它互译
AMS1117-1.8V 小板把排针上的 3.3V 变成 1.8V，专门喂给翻译器缺的那一侧电源
杜邦线各种公头/母头跳线，用来对接

电脑上还要：能 SSH 登录板子、能交叉编译（与第一章相同的工具链）。程序目录：chapters/ch03/code/temp-fan/。

荔枝派排针怎么认

板子侧面那排竖针，叫 20 针排针（两排，每排 10 根）。本课只用其中几根。

官方文档写明：这些信号脚的「高电平」大约是 1.8V（不是常见的 3.3V）。所以传感器、继电器信号要经过翻译器。排针上的 电源脚另有：

排针上的字本课用途
5V给继电器线圈供电；给风扇供电（经继电器）
3.3V给 DHT22、翻译器高压侧、1.8V 降压小板的输入
GND地线；所有模块的地都要连到这里（共地）
IO1_3本课默认：风扇/继电器信号脚
IO1_4本课默认：DHT22 数据线
插针方位图见仓库 chapters/ch03/manuals/lpi4a-wiki-assets/io_map.png。以文档图为准；早期板丝印可能不准。

程序里怎么对应：官方示例用设备节点 /dev/gpiochip1，其中 第 3 路对应 IO1_3，第 4 路对应 IO1_4。下面源码宏已按此写好。

接线总表（照抄）

先认清每块板子上的丝印文字（印在绿/黑板上的字母），再接线。丝印若和表不完全一致，以实物丝印为准，逻辑按下表「角色」对应。

A. 供电与地（先接这些）

从到说明
排针 GND所有模块的 GND / V- / 「-」漏接共地 = 程序在跑、硬件没反应
排针 3.3VDHT22 的 VCC；翻译器的 VCCB；1.8V 小板的 IN3.3V 侧电源
排针 5V继电器的 V+ / VCC继电器线圈要 5V
1.8V 小板的 OUT翻译器的 VCCA给翻译器低压侧供电
翻译器的 OE接到高电平（常接到 VCCA 一侧的高，或按小板说明短接使能）OE = 使能脚；不拉高时翻译器不工作

B. 信号线（经翻译器）

板子侧（约 1.8V）翻译器模块侧（约 3.3V）
排针 IO1_3A1 ←→ B1继电器信号脚 IN / X1
排针 IO1_4A2 ←→ B2DHT22 的 DAT / DOUT / OUT

翻译器：A 排靠近低压（板子），B 排靠近高压（模块）。同一编号 A1 与 B1 是一对。

C. 风扇电源（走继电器触点）

接法说明
排针 5V → 继电器 NO（常开）平时断开
继电器 COM（公共）→ 风扇红线继电器吸合后红线才有电
风扇黑线 → 排针 GND负极回地
继电器上的跳线帽拨到 H（高电平有效）。程序写「开」时给高电平，才和说明书一致。拨在 L 时，用 3.3V 信号 + 5V 供电容易出现「怎么写都像在吸合」的怪现象。 
排针 5V ──► [继电器 NO]──(吸合时接通)──[COM]──► 风扇红线
排针 GND ─────────────────────────────────────► 风扇黑线
板子 IO1_3 ──► 翻译器 ──► 继电器 IN
板子 IO1_4 ──► 翻译器 ──► DHT22 DAT

步骤 1：接线

  1. 板子断电（拔电源更安心）。
  2. 按上一节总表，先接完所有 GND，再接 3.3V / 5V，再接信号线，最后接风扇。
  3. 确认继电器跳线帽在 H。
  4. 确认没碰到 Fan Power 白色插座。
  5. 再上电。

本步成功判据：各模块电源指示灯（若有）亮；继电器未触发时风扇不转；你能指着线说出「这根是地 / 这根是风扇信号」。

步骤 2：改程序里的数字

打开 chapters/ch03/code/temp-fan/main.c 顶部宏。本课默认已按「IO1_3 / IO1_4」写好；若你脚位不同，只改这里的数字：

#define GPIO_CHIP_PATH   "/dev/gpiochip1"
#define DHT_LINE         4   /* IO1_4 */
#define FAN_LINE         3   /* IO1_3 */
#define T_HIGH           28.0f
#define T_LOW            26.0f
#define SAMPLE_MS        2000
#define SIMULATE_SENSOR  1   /* 先用假温度；真传感器再改 0 */

人话：SIMULATE_SENSOR 为 1 时，程序内部假装温度慢慢升降，不读 DHT——用来先验证「阈值逻辑 + 风扇会转」。接好真传感器后再改成 0，重新编译上传。

步骤 3：假温度模式跑通风扇

$ cd chapters/ch03/code/temp-fan
$ make clean && make
$ file temp-fan
# 成功：输出里有 RISC-V / UCB RISC-V；若是 x86-64 说明没用交叉工具链
$ scp temp-fan 用户名@板子IP:~/
$ ssh 用户名@板子IP './temp-fan'

把 用户名@板子IP 换成你第一章能登录的账号和地址。

预期：

  • 启动有 SIMULATE_SENSOR=1 一类提示，并打印 T_HIGH / T_LOW。
  • 周期性打印 [INFO] temp=… hum=… fan=ON|OFF。
  • 假温度升过上限 → [INFO] fan ON，继电器吸合（常有「咔」一声或工作灯亮），风扇转。
  • 假温度降过下限 → [INFO] fan OFF，风扇停。

停程序：在 SSH 里按 Ctrl+C。

本步成功判据：能演示超温开 / 降温关（假温度即可）。

步骤 4：接真传感器

  1. 确认 DHT22 的 VCC / GND / DAT 已按总表接好（DAT 经翻译器到 IO1_4）。
  2. 把 SIMULATE_SENSOR 改成 0，重新 make、scp、SSH 运行。
  3. 用手掌捂住 DHT22 白壳，或靠近热源（注意安全，别烫坏），看温度数字上升。
  4. 超过 T_HIGH（默认 28）应开风扇；离开捂热、温度降到低于 T_LOW（默认 26）应关风扇。
室温若已高于 28，可把 T_HIGH / T_LOW 临时改成「比当前室温高一点 / 低一点」的两个数，方便验收；改完重新编译。

本步成功判据：真温度能打印；捂热开、降温关可演示。

步骤 5：故意松线看报错

程序在跑、真传感器模式时，轻轻拔松 DHT22 的 DAT 线（或翻译器上对应那根），不要拔电源乱拔一气。

预期：终端出现类似 [ERR] DHT22 read failed … 的一行；进程不退出，继续下一轮采样。插回线后应恢复打印温度。

本步成功判据：有 [ERR] 且程序还在跑。

步骤 6：只读核对（加深理解，固定纪律）

devmem 是一个小工具：按「物理地址」去读（或写）内存里映射出来的外设寄存器。本课纪律：只用它读，禁止用它写——写会和系统里的 GPIO 驱动抢硬件，行为不可预期。

目的：当你用程序（经翻译器）把风扇脚置高时，内核驱动会改某个寄存器里的某一位。你在改之前读一次、改之后再读同一地址，若对应那一位从 0 变 1（或反过来），就证明「程序 → 内核 → 寄存器位」这条链真实发生过。

  1. 向当堂材料或讲义深入理解节，拿到本板要用的地址和第几位（不要猜）。
  2. 风扇仍为 OFF 时：sudo devmem 0x地址 32，抄下读数，标「改前」。
  3. 让程序进入 fan ON（或单独把该脚置高）。
  4. 再读同一地址，标「改后」，指出哪一位翻转。
  5. 全程不要带「写入数值」参数的 devmem 命令。

若当堂尚未下发地址，本步可记为「待补地址」；前五步验收仍可先完成。
