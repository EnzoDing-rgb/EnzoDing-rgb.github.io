本章实验 · 串口命令温控

自动控温同时敲 status / set · 接讲义 4.1–4.3 · chapters/ch04/code/serial-thermo/

说明

讲义 4.1–4.3 已讲清串口字节进出、命令表（把字符串命令名映射到处理函数）以及用 select 同时等待「该采样了」和「你打了一行字」。本实验在现成脚手架上补全命令处理与主循环，并在荔枝派 4A 上演示两件事同时成立：

  • 程序自动按阈值开/关风扇（与第三章温控闭环同一套阈值思路）；
  • 你在终端敲 status / set，立刻得到回复，且新阈值马上参与后续控温。
前置：已读讲义 4.1–4.3；第三章温控相关接线仍可用（DHT22 / 继电器 / 风扇）；RuyiSDK 交叉工具链与 SSH 可用。工程目录：chapters/ch04/code/serial-thermo/（源码 main.c，说明见同目录 README.md）。
你要改的代码：只动 main.c 里标了 TODO 的三处——cmd_status / cmd_set、命令表 g_cmds[]、以及 main_loop 里的 select 主循环。不要删掉已有的采样（sample_and_control）与风扇逻辑。默认 USE_STDIO=1 时，命令从 SSH 终端的标准输入读入，不必另接物理串口线即可联调。

步骤

1. 核对宏与接线

打开 chapters/ch04/code/serial-thermo/main.c，确认 GPIO_CHIP_PATH、DHT_LINE、FAN_LINE 与课程接线表一致。首次演示建议保持 SIMULATE_SENSOR=1（假温度缓缓升降，方便验证阈值与命令）；接好真 DHT22 后再改为 0 并重新编译。

本步成功判据：宏与接线一致；你知道当前是模拟传感器还是真传感器。

2. 补全命令处理与命令表

命令表是一张「名字 → 函数指针」的列表：收到一行输入后，先拆出命令名，再在表里查找并调用对应函数。脚手架已提供 dispatch_command 查找逻辑，你只需实现函数并把条目挂进表。

  1. 实现 cmd_status
    从全局状态 g_st 打印：最近温度、湿度、风扇 ON/OFF、t_high / t_low。
    若尚未采到有效样本（例如 has_sample 为 0），也要明确打印「尚无采样」之类提示，而不是输出未初始化的垃圾数。
    成功样子：敲一行 status 后，终端立刻出现完整状态行，进程不退出。
  2. 实现 cmd_set
    解析参数形态：high <数> 或 low <数>（用户敲的整行是 set high 30 / set low 25；表项名字是 set，后面的词作为 args 传入）。
    写入 g_st 前必须校验 low < high；不满足时打印 [ERR] 并拒绝修改，进程不退出。
    成功样子：set high 30 后再 status，上限已变为 30；故意设成 low >= high 时被拒绝并有错误提示。
  3. 注册到 g_cmds[]
    取消注释或填写 { "status", cmd_status } 与 { "set", cmd_set }，表尾保持 { NULL, NULL } 结束。
    成功样子：未知命令（如 foo）打印 [ERR] unknown command；已知命令能进到你的处理函数。

3. 补全 select 主循环（说清要做什么）

脚手架里的占位循环是「先采样，再 usleep」：睡眠期间你敲的字不会被处理，两边不能同时工作。你要改成用 select 一次等待两件事：

  1. 准备等待集合与超时
    把命令输入描述符 cmd_fd（USE_STDIO=1 时即标准输入）放进 fd_set；把超时设为距离下一次采样还剩多少时间（初始可设为 SAMPLE_MS 毫秒）。
  2. 调用 select
    返回值含义（本课用法）：
    • 返回 0：超时到期 → 调用已有的 sample_and_control() 做一轮采样与阈值控温，然后重新装满等待时间；
    • 返回正数：有描述符可读 → 调用已有的 read_command_line() 读一行并查表执行；处理完后应扣减已经过去的时间，避免采样间隔越来越稀；
    • 返回负数：打印错误并退出循环（或按讲义约定处理）。
  3. 未知命令与错误
    查不到的命令打印简短 [ERR]，进程继续跑，不要 exit。

本步成功判据：控温日志仍按约 SAMPLE_MS 周期出现；你在两次采样之间敲 status，回复几乎立即出现，而不是等到下一次睡眠结束才回。

4. 交叉编译并部署到板端

$ cd chapters/ch04/code/serial-thermo
$ make clean && make
$ file serial-thermo
# 应为 RISC-V ELF
$ scp serial-thermo user@board-ip:~/
$ ssh user@board-ip './serial-thermo'

启动后应看到阈值、采样周期，以及「可输入 status / set …」一类提示（具体文案以你程序为准）。

5. 联调顺序（固定，全部要做）

  1. 启动后观察至少两轮周期日志（温度 / 风扇 / 阈值），确认自动采样在跑。
  2. 敲 status，核对打印字段完整（温度、湿度、风扇、上下限）。
  3. 敲 set high 30，再敲 status，确认上限已变。
  4. 故意敲非法组合（例如把 low 设得不低于 high），确认被拒绝并有 [ERR]。
  5. 在模拟升温或捂热过程中，确认风扇仍按阈值动作；同时连续敲几次 status，确认打字回复不卡住采样（日志不会长时间停更）。
