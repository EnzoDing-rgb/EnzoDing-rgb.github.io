CoreMark 上板跑分

目标：把讲义 1.3 编好的 CoreMark 传到板端，运行并记下分数。分数高低不是评分重点；走通「编译 → 传输 → 运行 → 读数」才是。

步骤

  1. 在主机确认产物为 RISC-V ELF（讲义已做过则保留原日志即可）： 
    $ file coremark.exe
# 应含 UCB RISC-V / RISC-V 字样
  2. SCP 上传到板端（用户名与 IP 换成你的板子）： 
    $ scp coremark.exe user@board-ip:/tmp/
$ ssh user@board-ip 'sha256sum /tmp/coremark.exe'
# 与主机 sha256sum 对照，确认传输完整
  3. SSH 登录板端，赋予执行权限并运行： 
    $ ssh user@board-ip
$ chmod +x /tmp/coremark.exe
$ /tmp/coremark.exe
  4. 从输出中抄录 CoreMark 分数（常见字段为 iterations/sec 或 CoreMark 总分）。把完整终端输出另存为日志文件或截图。
排错提示：若板端报「Exec format error」或无法执行，先回主机用 file 核对是否误用了主机 GCC；再核对 SCP 路径与权限。命令细节见讲义 1.2 / 1.3。

必记字段

字段你的记录
产物文件名
主机 file 一行摘要
板端路径
CoreMark 分数
运行日期 / 板卡型号

通道与工具链自证

在完成跑分的同时，用下列命令自证开发通道与工具链可用。每条保留一行真实输出即可。

板端：SSH 与架构

$ ssh user@board-ip 'hostname; uname -m'
# uname -m 应为 riscv64

主机：工具链三元组

$ riscv64-unknown-linux-gnu-gcc -dumpmachine
# 应为 riscv64-unknown-linux-gnu
