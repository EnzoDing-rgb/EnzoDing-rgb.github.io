说明

讲义已说明单线程互堵、三线职责与锁 / TSan。本实验交付两份工程：

  • chapters/ch06/code/tri-thread/ — 荔枝派 4A 产品骨架（采样、阈值风扇、MQTT）；
  • chapters/ch06/code/race-demo/ — 电脑上跑的强制竞态小程序 + TSan。
前置：讲义 6.1–6.3；第五章程控灯与 Broker；温控接线可用。工具链：板上用 RuyiSDK 交叉编译；TSan 用主机 gcc。

步骤

A. 电脑：race-demo + TSan（先做，固定顺序）

  1. cd chapters/ch06/code/race-demo && make && ./race-demo — 记录最终计数（应明显小于期望，证明丢更新）。
  2. 执行 sudo sysctl vm.mmap_rnd_bits=28（让本机 ThreadSanitizer 稳定跑通）。
  3. make tsan && ./race-demo — 保存 ThreadSanitizer 报告全文，标出它指出的两行源码。

B. 板上：tri-thread

  1. 按接线表改 GPIO / Broker 宏；补全三线程循环（采集写温度、控制判阈值、通信跑 MQTT）。
  2. 先不加锁（脚手架提供 USE_LOCK 0）：跑一段时间，记录错乱现象（温度跳变、风扇状态与温度矛盾等）。
  3. 改为 USE_LOCK 1，所有共享读写进锁；确认行为正确。
  4. 交叉编译部署： 
    $ cd chapters/ch06/code/tri-thread
$ make clean && make
$ scp tri-thread user@board-ip:~/
$ ssh user@board-ip './tri-thread'
  5. 验证：风扇按阈值动作；MQTT 下行/上行仍工作。
  6. Ctrl+C：风扇关断后进程退出（干净停）。
